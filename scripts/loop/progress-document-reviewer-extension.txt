Feature: document-reviewer-extension

## Codebase Patterns

- Render Mermaid fences in a second pass after markdown HTML insertion, keep source in a single hidden node, and sanitize parsed SVG before DOM insertion so fallback remains local and safe.
- Keep extension command validation non-blocking and race-safe with `fs.promises.lstat(...)` + explicit `ENOENT` messaging.
- Use a shared launcher helper in `extensions/lib/` and surface a fallback command in the editor when browser launch fails.
- For local review APIs, enforce `127.0.0.1` binding plus workspace-root containment using realpath checks to block symlink escape paths.
- Serve reviewer UI via tokenized HTML template (`__REVIEW_BOOTSTRAP__`, `__REVIEW_ASSET_BASE_PATH__`) and inject session bootstrap JSON server-side.
- For reviewer keyboard UX, keep a single `keydown` dispatcher and ignore events from editable targets to avoid future comment-input conflicts.
- Split input handling into a pure keymap resolver plus a DOM selection controller; this keeps keyboard behavior unit-testable while reducing noisy UI state updates.
- For threaded-review panels rendered with `innerHTML`, persist per-thread reply drafts/errors in keyed maps so rerenders do not wipe in-progress reviewer input.
- For local review comment APIs, validate request shape early (`payload` must be JSON object + content-type JSON) and reject unsupported classification fields consistently for both thread creation and replies.
- Persist comments via a dedicated sidecar repository keyed by a stable path hash (`.review/<docHash>.comments.json`) and normalize legacy anchors (`quote` -> `exact`) before running deterministic re-anchor + stale detection.
- For End Review UX, keep clipboard/export behavior in a dedicated controller that emits explicit states (`copied`, `manual-copy`, `empty`, `error`) so app-level status updates stay simple and fallback text handling remains deterministic.
- Protect session-scoped local APIs with an ephemeral per-session token (`x-review-session-token`) and require the frontend bootstrap payload to include the token for every fetch helper call.
- Keep browser-launch failure recovery text in a dedicated helper so remote/headless guidance (manual URL + SSH tunnel hint) stays consistent and testable.
- In SVG sanitizers, always lower-case both the inspected tag name and denylist entries (`foreignobject`) to avoid case-mismatch bypasses.
