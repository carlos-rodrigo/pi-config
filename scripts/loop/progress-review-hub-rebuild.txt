# Loop Progress Log

Started: 2026-02-27
Feature: review-hub-rebuild

## Codebase Patterns

- `loadManifest()` should be the single normalization gate for schema evolution: interpret missing `schemaVersion` as v1, normalize comments, and always return v2 runtime shape.
- Degrade corrupt per-comment anchors non-fatally (drop `anchor`, keep comment), but fail fast on unsupported top-level schema versions.
- `createReviewServer(bridge?)` accepts an optional `ReviewRuntimeBridge` — defaults to `createNoOpBridge()` for backward compatibility.
- Service classes use constructor injection with accessor functions (`getManifest`, `getReviewDir`) to avoid stale state.
- `import type` is erased at runtime by tsx/esbuild — always include a value import when testing module existence.
- Use `type X = A | B` not `interface X { } | { }` — esbuild doesn't support the latter syntax.
- Cache source-derived payloads by `sourceHash` — pattern used by both `getVisualHtml` and `/visual-model`.
- `RenderSection` from `/visual-model` is the frontend's single source of truth for section identity.
- Shell layout uses slot pattern: `ReviewShell` accepts `tocRail`, `commentRail`, `bottomBar`, `statusBar` as ReactNode props.
- Mobile drawers use controlled `Sheet` with `open`/`onOpenChange` — triggers live in ShellHeader, content in ReviewShell.
- `react-markdown` remarkPlugins must be a module-level stable reference (not inline `[remarkGfm]`). Don't use `as const` — Pluggable[] type is mutable.
- Memoize individual section blocks with `memo()` to avoid re-rendering all sections on active section change.
- `useVisualModel(token)` hook fetches from `/visual-model` — returns `{ sections, isLoading, error }`.
- Anchor capture: pure `buildAnchorFromSelection()` in `lib/anchor/capture.ts` (testable), DOM hook `useSelectionAnchor` in hooks layer.
- Server validates anchor shape via `isValidAnchorShape()` before persisting. `loadManifest` normalization handles corrupt anchors on read.
- `DocumentViewportHandle.getContainerRef()` exposes container for scoped DOM event listeners.
- `resolveAnchor()` in `lib/anchor/reanchor.ts` is pure: text in → resolution out (exact/reanchored/degraded).
- `useReanchorMap(comments, sections)` returns `Map<commentId, AnchorResolution>`. Uses section text Map for O(1) lookup.
- `HighlightLayer` renders quote chips below sections. Highlight chips > inline marks for react-markdown content.
- `highlightsBySectionId` groups `HighlightEntry[]` by sectionId for efficient per-section rendering.
