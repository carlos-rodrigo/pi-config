## Codebase Patterns

- `loadManifest()` should be the single normalization gate for schema evolution: interpret missing `schemaVersion` as v1, normalize comments, and always return v2 runtime shape.
- Degrade corrupt per-comment anchors non-fatally (drop `anchor`, keep comment), but fail fast on unsupported top-level schema versions.
- `createReviewServer(bridge?)` accepts an optional `ReviewRuntimeBridge` â€” defaults to `createNoOpBridge()` for backward compatibility.
- Service classes use constructor injection with accessor functions (`getManifest`, `getReviewDir`) to avoid stale state.
- `import type` is erased at runtime by tsx/esbuild â€” always include a value import when testing module existence.
- Use `type X = A | B` not `interface X { } | { }` â€” esbuild doesn't support the latter syntax.
- Cache source-derived payloads by `sourceHash` â€” pattern used by both `getVisualHtml` and `/visual-model`.
- `RenderSection` from `/visual-model` is the frontend's single source of truth for section identity.
- Shell layout uses slot pattern: `ReviewShell` accepts `tocRail`, `commentRail`, `bottomBar`, `statusBar` as ReactNode props.
- Mobile drawers use controlled `Sheet` with `open`/`onOpenChange` â€” triggers live in ShellHeader, content in ReviewShell.
- `react-markdown` remarkPlugins must be a module-level stable reference (not inline `[remarkGfm]`). Don't use `as const` â€” Pluggable[] type is mutable.
- Memoize individual section blocks with `memo()` to avoid re-rendering all sections on active section change.
- `useVisualModel(token)` hook fetches from `/visual-model` â€” returns `{ sections, isLoading, error }`.
